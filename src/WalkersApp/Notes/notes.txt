Design pattern that seperates Data access layer  from the appliation
Provides interface without exposing implementation.
Helps create abstraction.
Repository class is responsble for doing the CRUD operations.
By injecting the interface, into the controller - the controller will not know whether its a SQL db or Mongo
db but just uses the functions from the interface to do CRUD.
BY this way the appliation becomes highly scalable.

Benefits:
1. Decoupling
2. Consistency
3. Performance
4. Mutiple data sources.

If there are multiple DbContext files, the DbContextOptions should not be non-generic.If not, this 
is the exception we get.
"Exception User-Unhandled

System.InvalidOperationException: The DbContextOptions
passed to the NZWalksDbContext constructor must be a DbContextOptions<NZWalksDbContext>.
When registering multiple DbContext types, make sure that the constructor for each context 
type has a DbContextOptions<Context> parameter rather than a non-generic DbContextOptions parameter.
"
